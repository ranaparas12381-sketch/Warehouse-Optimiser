# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository, please do not open a public issue with exploit details.

Instead:

1. Prepare a concise description of the issue.
2. Include reproduction steps, impact, and any affected files or routes.
3. Share the report privately with the repository owner through the available GitHub contact channels.

## Scope

Security-sensitive areas in this project include:

- deployment configuration
- HTTP environment routes
- dependency management
- container packaging

## Handling

Reported issues should be triaged, reproduced, and fixed before public disclosure where practical.
